Dartmouth BASIC (1964)

Kemeny & Kurtz. Every statement needs a line number. Flow control is pure GOTO. No subroutines.

Line numbers required GOTO only No subroutines No string ops
Program — sum of 1 to N, print if divisible by 3
10 REM SUM INTEGERS 1 TO N, PRINT MULTIPLES OF 3 20 LET N = 20 30 LET S = 0 40 LET I = 1 50 IF I > N THEN 100 60 LET S = S + I 70 LET R = S - 3 * INT(S / 3) 80 IF R = 0 THEN 90 85 GOTO 95 90 PRINT "SUM=", S, " AT I=", I 95 LET I = I + 1 96 GOTO 50 100 PRINT "DONE" 110 END
  • LET required for assignment
  • IF/THEN targets a line number only — no ELSE
  • Loops built entirely from IF + GOTO
  • MOD doesn't exist — integer trick needed
  • All variable names one letter (or letter+digit)

BASIC with GOSUB (late 1960s)

GOSUB/RETURN added subroutine calls. Still fully line-numbered. FOR/NEXT loops appear. Logic is cleaner but spaghetti persists.

Line numbers required GOSUB / RETURN FOR / NEXT Still GOTO-heavy
Same program — now using GOSUB for the print action
10 REM SUM 1 TO N, PRINT MULTIPLES OF 3 VIA GOSUB 20 LET N = 20 30 LET S = 0 40 FOR I = 1 TO N 50 LET S = S + I 60 IF (INT(S/3))*3 = S THEN GOSUB 200 70 NEXT I 80 PRINT "DONE" 90 END 200 REM SUBROUTINE: PRINT CURRENT SUM AND INDEX 210 PRINT "SUM="; S; " AT I="; I 220 RETURN
  • FOR/NEXT replaces manual loop counter GOTO
  • GOSUB jumps to a subroutine; RETURN comes back
  • Subroutines share global variables — no parameters
  • Subroutines identified only by line number
  • IF/THEN still targets a line or GOSUB — no ELSE yet

Microcomputer BASIC (1975–1983)

Microsoft BASIC, Applesoft, Commodore, TRS-80. Line numbers everywhere. Strings, INPUT, PEEK/POKE. The era of the magazine type-in.

Line numbers required String variables ($) INPUT statement PEEK / POKE ON/GOTO dispatch
Same logic — now interactive, with string output
10 REM MICROCOMPUTER BASIC - SUM & DIVISIBILITY 20 INPUT "HOW HIGH? ", N 30 S = 0 40 FOR I = 1 TO N 50 S = S + I 60 IF S MOD 3 = 0 THEN GOSUB 200 70 NEXT I 80 PRINT "ALL DONE!" 90 END 200 REM PRINT SUBROUTINE 210A$ = "SUM=" + STR$(S) + " AT I=" + STR$(I) 220PRINT A$ 230RETURN
  • LET now optional — plain assignment works
  • MOD operator available on most dialects
  • String variables use $ suffix; STR$() converts numbers
  • INPUT prompts the user at runtime
  • ON X GOTO / ON X GOSUB for dispatch tables
  • PEEK/POKE for direct memory access (hardware era)

Structured BASIC (1985–1990)

BASICA, GW-BASIC, then QuickBASIC. WHILE/WEND, THEN/ELSE, named SUBs and FUNCTIONs. Line numbers become optional. The spaghetti starts to untangle.

Line numbers optional WHILE / WEND IF / THEN / ELSE Named SUB / FUNCTION Local variables
Same program — named SUB, ELSE branch, no line numbers needed
REM QUICKBASIC-STYLE STRUCTURED BASIC DECLARE SUB PrintSum (s AS INTEGER, i AS INTEGER) DIM n AS INTEGER DIM s AS INTEGER INPUT "How high? ", n s = 0 FOR i = 1 TO n s = s + i IF s MOD 3 = 0 THEN CALL PrintSum(s, i) ELSE REM not a multiple — keep going END IF NEXT i PRINT "Done." SUB PrintSum (s AS INTEGER, i AS INTEGER) PRINT "Sum="; s; " at i="; i END SUB
  • Named SUB/FUNCTION with typed parameters
  • IF / THEN / ELSE / END IF blocks — multi-line
  • DIM for variable declarations with types
  • WHILE / WEND (and later DO / LOOP) for conditional loops
  • GOTO still works but is no longer necessary
  • SELECT CASE replaces ON/GOTO dispatch
  • Line numbers no longer required
  • GOSUB to a number is deprecated in favor of CALL

1990s BASIC (Visual Basic / QB era)

Visual Basic, QBasic, PowerBASIC. Full structured programming. Types, arrays, user-defined types. Line numbers gone. Reads like pseudocode.

No line numbers DO / LOOP TYPE records Arrays String functions
Same program — clean modern style, result in a string
FUNCTION RunningSum(n AS INTEGER) AS STRING DIM s AS INTEGER DIM i AS INTEGER DIM out AS STRING s = 0 out = "" FOR i = 1 TO n s = s + i IF s MOD 3 = 0 THEN out = out + "Sum=" + STR(s) + " at i=" + STR(i) + CHR(10) END IF NEXT i RunningSum = out END FUNCTION DIM n AS INTEGER INPUT "How high? ", n PRINT RunningSum(n) PRINT "Done."
  • FUNCTION returns a value — assigned by name
  • Fully local variables with DIM inside procedures
  • String concatenation with + operator
  • CHR() / ASC() / LEN() / MID() string library
  • SELECT CASE, DO WHILE / LOOP, DO / LOOP UNTIL
  • User-defined TYPE records (like structs)
  • GOTO essentially gone from well-written code
  • GOSUB completely replaced by CALL and FUNCTION

FreeBASIC (2004–present)

Full C-level power with BASIC syntax. Pointers, UDTs, operator overloading, namespaces, inline assembly. Compiles to native binaries.

Compiled to native Pointers UDT / TYPE Namespaces No line numbers
Same program — typed, procedural, native-compiled
' FreeBASIC: sum 1..n, collect divisible-by-3 results Type SumResult total As Integer atStep As Integer End Type Function CollectSums(n As Integer) As String Dim As Integer s = 0, i Dim As String out = "" Dim r As SumResult For i = 1 To n s += i If s Mod 3 = 0 Then r.total = s r.atStep = i out += "Sum=" & r.total & " at i=" & r.atStep & !"\n" End If Next i Return out End Function Dim As Integer n Input "How high? ", n Print CollectSums(n) Print "Done."
  • TYPE (UDT) as proper record structs
  • Compound assignment: +=, -=, *=
  • & string concatenation (vs + which can be ambiguous)
  • Escape sequences in strings with !
  • RETURN keyword to return a value from FUNCTION
  • Pointers, dynamic memory, inline ASM available
  • Compiles to x86/x86-64 native — no interpreter
  • Line numbers: completely gone
  • GOTO: available but virtually unused
  • GOSUB: gone — replaced entirely by SUB/FUNCTION
The GOTO problem. In 1964, GOTO wasn't bad style — it was the only branching tool that existed. The IF statement could only jump to a line number; there was no IF/THEN/ELSE block. So every decision tree was woven out of numbered lines and jumps, and any program of modest size became very hard to follow. GOSUB was a big deal. When GOSUB/RETURN arrived it let you think in terms of reusable actions, even though subroutines were still identified by line number, took no parameters, and shared all variables globally. It planted the seed of the idea that became named SUBs. The microcomputer era froze things. Microsoft BASIC shipped on nearly every home machine from 1975 through the mid-80s, and it was essentially the GOSUB-era dialect with strings and PEEK/POKE bolted on. A whole generation of programmers learned on it. QuickBASIC was the real structural shift. Named SUBs with typed parameters, IF/THEN/ELSE/END IF blocks, WHILE/DO loops — suddenly BASIC looked like Pascal. GOTO became unnecessary rather than unavoidable. FreeBASIC closes the gap with C. It keeps the BASIC readability while adding UDTs, pointers, compound assignment, native compilation, and inline assembly. The language you used for a CS elective class and the language you'd write embedded firmware in are the same language.
